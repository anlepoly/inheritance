BUG REPORT COMMENTS:
The current implementation of toSource in Rhino does not allow to transfer
Object.prototype.toSource to be transfered to other objects since it uses Java
inheritance to call a particular method and as such does not fit into prototype
pattern of JS. 

In the current SM shell I have:


js> var a = [1,2];
js> a.toSource = Object.prototype.toSource

function toSource() {
    [native code]
}

js> a.toSource()
({0:1, 1:2, toSource:function toSource() {[native code]}})

js> Object.prototype.toSource == Array.prototype.toSource
false


while the same session in Rhino gives:


js> var a = [1,2];
js> a.toSource = Object.prototype.toSource
function toSource() { [native code for Object.toSource, arity=0] }

js>  a.toSource()
[1, 2]

js>  Object.prototype.toSource == Array.prototype.toSource
true
Targeting 1.5R5


I committed the fix
